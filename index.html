How it works

Signal, filtered by your stack.

01

Describe your stack

List clouds, languages, packages, services, and keywords in YAML.

02

Poll trusted feeds

NVD, GitHub advisories, and MSRC updates are checked on a schedule.

03

Match and notify

Only items that match your stack are sent to Discord.

Setup

Run locally, on cron, or inside Docker.

1. Configure

cp config.example.yaml config.yaml
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/..."

2. Install

pip install -r requirements.txt

3. Run

python -m src.main --config ./config.yaml --once

Optional: Docker or cron

Run in Docker for a clean host, or cron for a lightweight schedule.

docker run \
  -v ./config.yaml:/app/config.yaml \
  -v ./state.json:/app/state.json \
  security-stack-notifier

# or cron (every 30 minutes)
*/30 * * * * cd /path/to/signl && /path/to/signl/.venv/bin/python -m src.main --config ./config.yaml --once \
  >> /path/to/signl/signl.log 2>&1

Config

Target what matters to your environment.

Example config.yaml

version: 1

mode: normal

always_page:
  - trufflehog
  - "auth bypass"

stack:
  cloud:
    - azure
    - aws
    - gcp

  languages:
    - python
    - javascript
    - typescript
    - powershell
    - go

  packages:
    npm:
      - axios
      - react
      - lodash
      - express
    pip:
      - requests
      - django
      - flask
      - pydantic
      - azure-identity
    go:
      - gin
      - cobra

  services:
    - entra-id
    - intune
    - azure-sentinel
    - kubernetes
    - docker
    - postgresql
    - redis

  keywords:
    - kerberos
    - ntlm
    - active directory
    - oauth
    - saml

notifications:
  discord:
    webhook_url: "${DISCORD_WEBHOOK_URL}"

settings:
  poll_interval_minutes: 15
  state_file: "./state.json"
  include_low_severity: false
  max_results_per_feed: 200
  request_timeout_seconds: 20
  user_agent: "security-stack-notifier/0.1"

Match logic

  • Direct package name match
  • Service name match
  • Keyword match
  • Language mention (contextual)
  • Cloud provider mention (contextual)

Noisy matches are avoided with token-aware matching for short words.

Alert Modes

Choose how noisy the alerts should be.

Alert Modes

( ) Quiet (*) Normal (default) ( ) Loud

Quiet: only wake us up for real fires. Normal: serious and relevant issues. Loud: everything relevant, immediately. Everything else is rolled into a digest.

Mode snippets

mode: quiet
# only high-confidence exploitation signals
mode: normal
# serious + relevant (default)
mode: loud
# page on any relevant alert
